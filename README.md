# 世界书读取

一个用于 **SillyTavern** 的前置世界书路由插件。

它不会替代 SillyTavern 原生世界书，而是在正式生成回复之前增加一层“前置 AI 路由”：

- 读取当前用户消息与最近上下文
- 读取当前角色卡 / 绑定世界书中的条目
- 先做本地候选召回
- 再由前置 AI 从候选中精选
- 将本轮真正相关的世界书内容注入到 prompt 中

这样可以减少无关条目注入，降低上下文浪费，让大型世界书在实际对话里更稳定地命中有效信息。

## 功能特性

- 生成前自动读取并筛选本轮相关世界书
- 支持角色卡内嵌 `character_book.entries`
- 支持角色 / 聊天 / 全局绑定世界书读取
- 默认跳过 `disabled` 条目
- 默认跳过 `constant` 条目，避免与原生常驻注入重复
- 本地关键词候选召回
- 前置 AI JSON 精选
- AI 失败时自动 fallback 到本地候选评分
- 支持独立路由模型（OpenAI-compatible）
- 支持可选读取 `MVU / stat_data`
- 支持查看：
  - 最近一次候选与选择结果
  - 发给前置 AI 的 Prompt
  - 前置 AI 的返回
  - 本轮最终注入文本
- 生成时世界书按钮下方流光提示
- 路由完成后显示本轮命中条目动画
- 支持前置 AI 重试次数与失败动画反馈

## 适用场景

适合下面这类卡或世界书：

- 世界书条目很多，原生关键词不够灵活
- 角色卡依赖大量背景设定、人物设定、地点设定
- 希望本轮只注入少量真正相关条目
- 希望把“候选召回”和“最终精选”拆开处理

## 安装方式

将本插件目录放到：

```text
SillyTavern/public/scripts/extensions/third-party/ai-worldbook-router
```

目录内应至少包含：

- `manifest.json`
- `index.js`
- `settings.html`
- `style.css`

放入后刷新 SillyTavern 页面，在扩展设置中启用 **世界书读取**。

## 基本使用

启用插件后，它会在每次正式生成前执行以下流程：

1. 收集最近聊天上下文
2. 读取当前可用世界书条目
3. 做本地候选召回
4. 调用前置 AI 精选候选
5. 将选中的条目拼成“本轮相关世界书”注入块
6. 再交给主模型生成正式回复

## 设置说明

### 核心项

- `启用前置 AI 世界书路由`  
  插件总开关。

- `最大候选条目数`  
  本地粗召回后最多保留多少条候选给前置 AI 判断。

- `最终注入条目数`  
  本轮最多注入多少条世界书。

- `最大注入字符数`  
  防止注入块过长，挤占主回复上下文。

- `扫描最近消息数`  
  候选召回和 AI 判断时使用的最近消息数量。

### 路由模型

- `启用独立模型`  
  打开后，前置 AI 路由会使用单独配置的模型，不占用当前主回复模型配置。

- `API URL / API Key / 路由模型`  
  用于独立路由模型调用。

- `AI 判定最大回复 tokens`  
  限制前置 AI 返回 JSON 的长度。

- `前置 AI 重试次数`  
  当前置 AI 调用失败或 JSON 解析失败时，自动重试的次数。

### 可选兼容

- `读取 MVU/stat_data（可选兼容）`  
  会尝试读取 Tavern Helper / MVU 的状态数据，并将其纳入本轮判断。

- `允许重复路由 constant 条目`  
  默认建议关闭。开启后，插件也会重新考虑 `constant` 条目。

## 调试面板

插件面板中可以直接看到：

- 最近一次候选数量、命中数量、注入长度、来源
- 最近一次选中的条目
- 发给前置 AI 的 Prompt
- 前置 AI 返回
- 本轮最终注入文本

如果你想确认“世界书到底有没有真的进 prompt”，直接看 **本轮最终注入文本** 即可。

## 动画反馈

为了不打断对话流程，插件不会使用额外弹窗提示。

当前交互反馈为：

- 路由进行中：世界书按钮下方出现流光
- 路由成功：命中的条目从世界书按钮位置弹出后回收
- 路由重试：抛出 `🔄`
- 路由最终失败：抛出红色 `×`，并出现红色闪烁提示

## 兼容说明

- 本插件基于 SillyTavern 的 `generate_interceptor`
- 正常情况下会在主生成前完成
- 如果同时使用其他前置插件，只要对方也正确 `await` 自己的拦截器流程，就不会抢跑
